Interaction Design

课堂与助教的协作流程

核心原则：不改原课堂主流程，只在旁边开一条“助教支线”，结束后再合并回主课堂。

02 / 05
User Flow
连接只是待命，不暂停课堂
真正切入助教模式才暂停课堂
侧会话内容通过 brief 合并回主线
State Ownership
`idle`课堂正常，助教可连接但不抢占课堂
`handRaiseActive`请教助教中，lecture / discussion callbacks 全部 gated
`briefing`调用 `/api/voice-brief` 生成课堂可消费内容
`handoff`把 brief 走原 `sendMessage` / `handleUserInterrupt` 路径喂回课堂

UI 语义也做了重构

连接助教
开始请教助教
结束请教，回到课堂
课堂已暂停 / 课程继续中
助教浮窗可拖拽
延迟显示移除

Boundary Diagram
Product-Level Outcome
实时语音更像“助教”不是老师，不和课堂主讲人格竞争
课堂主线更稳定原多智能体逻辑基本不动，只接受 brief 输入
后续扩展面更宽支持多 realtime backend、个性 prompt、服务端 relay
上下文相关性更强server 可吃 system_prompt 与动态 context，不再像通用客服
